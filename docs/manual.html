章 1|紙娃娃系統

1-1 遊戲內換裝(零程式)

開啟遊戲(本機 npm run dev 後 http://localhost:5173/,或線上 demo),右上面板四個插槽,點色塊即時換裝:

插槽目前選項
髮色原色 / 金 / 粉 / 銀
上衣原色 / 紅 / 藍 / 綠
褲子原色 / 棕 / 綠
帽子無 / 棒球帽

換裝狀態跟著角色實例走:切換場景(進出房子)不會掉裝。

▲ 示範村莊:三棟房子,各自門前石徑與出入口(assets/scenes/outdoor.json)

1-2 原理:角色 = 一疊對齊的 sprite sheet

角色由多層 AnimatedSprite 疊成:底層 char-body,上面依序疊 pants → shirt → hair → hat overlay。每層都是兩張 4×4 sprite sheet(-walk 8fps / -idle 4fps),與 body 逐像素對齊,所以走路、待機時整套一起動。

程式入口:src/main.ts 的 slotDefs 定義插槽與變體;換裝呼叫 player.setOverlay(manifest, slot, name | null)(null = 脫掉)。UI 只會列出 sheet 檔案真的存在的變體。

1-3 加一個新衣物變體(三步)

  1. 生成 overlay sheet(髮色 / 衣色一律程式化換色,不用生圖模型畫——對齊不可控,專案已兩敗): 
    # 髮色(色名:blonde / pink / silver,要 walk+idle 各一張)
python3 tools/make-hair-overlay.py assets/raw/char-body-walk_sheet.png \
        assets/raw/char-hair-blonde-walk_sheet.png blonde

# 上衣/褲子(part:shirt / pants;shirt 色:red|blue|green,pants 色:brown|green)
python3 tools/make-clothes-overlay.py assets/raw/char-body-walk_sheet.png \
        assets/raw/char-shirt-red-walk_sheet.png shirt red
    全新形狀的配件(如帽子)才走 codex 生圖 + tools/make-hat-sheets.py 逐幀貼位。
  2. 註冊到 assets/manifest.json(walk / idle 各一個 entry): 
    "char-shirt-red-walk": { "sheet": "raw/char-shirt-red-walk_sheet.png",
  "grid": [4, 4], "fps": 8, "kind": "character-layer" },
"char-shirt-red-idle": { "sheet": "raw/char-shirt-red-idle_sheet.png",
  "grid": [4, 4], "fps": 4, "kind": "character-layer" }
  3. 加進 src/main.ts 的 slotDefs 對應插槽的 variants: 
    { slot: 'shirt', title: '上衣', none: '原色',
  variants: [['紅', 'char-shirt-red'], ['藍', 'char-shirt-blue'], …] }
    重新整理頁面,右上面板就會多出新色塊。
要開全新插槽(如「鞋子」)也一樣:slotDefs 加一組 { slot, title, none, variants } 即可,渲染順序在 src/player.ts 的層序決定。

章 2|布置一個房間

2-1 場景即資料:JSON 欄位

每個場景是 assets/scenes/<名字>.json 一個檔。以 office.json 為例:

欄位意義
name場景識別名(出入口 to 就填這個)
size地板世界尺寸 px,如 {"w":1500,"h":900}
wallHeight上牆高度 px(室內 130;室外沒牆給 60 當留白)
floorTile地磚一格的邊長 px
floor / wall地板 / 牆素材名;wall 可省略 = 無牆(室外)
spawn直接載入此場景時的角色出生點
objects家具清單:{"asset":"素材名","x":750,"y":300},可選 z
exits出入口清單(見章 3)

移動家具 = 改一行座標;新增家具 = 加一行。素材名總表在 assets/manifest.json;想先看素材長相,開 http://localhost:5173/#preview 全素材動畫預覽。

▲ 布置好的房間範例:assets/scenes/office.json(6 組桌椅、廚房區、書櫃)

2-2 用編輯模式拖出你要的格局

  1. 右上面板點「▶ 進入編輯模式」。
  2. 點任一家具 → 綠色高亮;按住拖曳,y-sort 遮擋與碰撞箱即時跟著動。
  3. 排好後點「複製場景 JSON」:內容進剪貼簿(權限被擋時自動改成下載檔案)。
  4. 貼回 assets/scenes/<名字>.json 存檔,重新整理就是新格局。

▲ 編輯模式:按鈕變「■ 編輯中(點家具拖曳)」,多出「複製場景 JSON」;點中的家具會綠色高亮

2-3 三個進階欄位

  • flat 素材(地毯 rug-red、石徑 stone-path):鋪在地上、永遠不遮擋角色,直接放 objects 即可(素材在 manifest 已標 "flat": true)。
  • z 排序覆寫:想把小物疊在檯面上,給大的 z,如 office 的咖啡機 {"asset":"coffee-machine","x":1180,"y":-18,"z":95} 疊在流理臺上。
  • 碰撞箱:定義在 manifest 的素材 collider,家具放進場景自動生效,不用另外設。

章 3|做一張多房子小地圖(房子可進出)

3-1 機制:exits 出入口

場景 JSON 的 exits 是一列「感應區」:角色碰撞箱與 zone(中心式 AABB:x,y 是中心點)重疊的那一幀,就切到 to 場景、角色落在 spawn。一個場景可放多個出入口,互不干擾。

"exits": [
  { "zone": { "x": 350,  "y": 525, "w": 90, "h": 30 }, "to": "cabin",
    "spawn": { "x": 450, "y": 500 } },
  { "zone": { "x": 1450, "y": 525, "w": 90, "h": 30 }, "to": "storage",
    "spawn": { "x": 400, "y": 460 } }
]

3-2 從零建村莊 step-by-step

  1. 開室外場景 assets/scenes/outdoor.json:floor 用 grass-tile、不給 wall、size 給大一點(示範村莊 1800×1100)。
  2. 放房子:{"asset":"house-exterior","x":900,"y":500}。房子碰撞箱 330×60、中心在 (x, y-30),角色會被擋在牆外、只留門口通道。
  3. 房子門前放出入口:zone 放在門口正下方 {"x":房子x,"y":525,"w":90,"h":30},to 填室內場景名。想加第 2、3 棟,就再放一組 house-exterior + exit(x 換位置),to 指到不同室內場景。
  4. 建室內場景(如 cabin.json):floor-wood + wall-top,家具照章 2 擺;底部放一個回程出入口: 
    "exits": [ { "zone": { "x": 450, "y": 590, "w": 170, "h": 30 },
  "to": "outdoor", "spawn": { "x": 350, "y": 620 } } ]
  5. spawn 幾何守則(最容易踩的坑):落點必須在目的地任何 zone 的觸發範圍之外,不然一進場立刻被彈回去。觸發半徑約 = zone 半高 + 角色半高(≈37px),落點離 zone 中心至少 70px 最保險——示範村莊全部用「zone y=525、門前落點 y=620」這組數字。
  6. 驗收:實際走進、走出每一棟房子,確認換裝還在、沒有一進場就被彈走。

▲ 第三棟房室內(assets/scenes/storage.json):同素材、另一種格局

3-3 對照實例

repo 裡的示範村莊就是照上面步驟做的,直接抄:

  • assets/scenes/outdoor.json — 室外 1800×1100,三棟 house-exterior(x=350 / 900 / 1450,y 全是 500),三個門前 exit 各連 cabin / office / storage。
  • assets/scenes/cabin.json — 900×600 小屋,底部 exit 回 outdoor 的 (350, 620)。
  • assets/scenes/storage.json — 800×550 儲物間,底部 exit 回 outdoor 的 (1450, 620)。
  • assets/scenes/office.json — 1500×900 辦公室,底部 exit 回 outdoor 的 (900, 620)。

首載素材約 34MB(全 sprite sheet 動畫),線上第一次開比較慢,屬已知現況。
